# Anthropic Client

A real backend adapter over the Anthropic SDK (`@anthropic-ai/sdk`), wired to the same `AnthropicClient` DI token the fake implementation already uses, so any feature that calls the Messages API for real injects `AnthropicClient` and gets the real client in real mode / the fake in fake mode with no per-feature branching.

## Interface

- **`AnthropicClient`** (`backend/src/shared/anthropic-client/anthropic-client.ts`) — the abstract-class DI token: `createMessage(params: AnthropicMessageParams, betas?: string[]): Promise<AnthropicMessage>`, `streamMessage(params, betas?: string[]): AsyncIterable<AnthropicStreamEvent>`, and `uploadFile(bytes: Buffer, mediaType: string): Promise<{ id: string }>`, the first two typed directly against `@anthropic-ai/sdk`'s own `Message` / `MessageCreateParams` / `RawMessageStreamEvent` types (`AnthropicMessage` / `AnthropicMessageParams` / `AnthropicStreamEvent` are this file's own aliases for those). Lives at this build-included shared location — not under `backend/src/testing/` — because a production module needs to bind a real class to it; `FakeAnthropicClient` (see [`test-doubles.md`](test-doubles.md)) stays under `testing/` and imports the token from here. `betas` names the SDK's beta-feature flags (e.g. `files-api-2025-04-14`) a request needs — pass it whenever the request relies on a field that only exists on the beta Messages types, such as a Files-API `file_id` document/image source (the stable `DocumentBlockParam.source` union has no `type: 'file'` member at all); omitting it, or passing an empty array, is the exact same behavior every non-beta caller already gets.
- **`RealAnthropicClient`** (`backend/src/shared/anthropic-client/real-anthropic-client.ts`) — constructs `new Anthropic({ apiKey: config.anthropicApiKey })` via `AppConfigService` (no `maxRetries` override, unlike [`key-health.md`](key-health.md)'s deliberately-fast check — a real user-facing call gets the SDK's normal retry behavior). `createMessage()` calls `client.messages.create({ ...params, stream: false })`; `streamMessage()` calls `client.messages.create({ ...params, stream: true })` and yields the SDK's own raw stream events unmodified. Whenever `betas` is non-empty, both route to `client.beta.messages.create({ ...params, betas })` instead (same request path, `POST /v1/messages?beta=true`, not a separate endpoint), casting the beta response back to the stable `AnthropicMessage`/`AnthropicStreamEvent` aliases at that one boundary. `uploadFile()` wraps `bytes` via the SDK's own `Anthropic.toFile(bytes, undefined, { type: mediaType })` helper (the installed SDK's `Uploadable` type requires a `File`/`Response`/`fs.ReadStream`-like value — a plain `Readable` doesn't satisfy it) and calls `client.beta.files.upload({ file, betas: ['files-api-2025-04-14'] })`, returning `{ id: uploadResult.id }`. Any thrown Anthropic SDK error (auth failure, rate limit, network error, etc.), from any of the three methods, is caught and rethrown as [`api-error-handling.md`](api-error-handling.md)'s `ExternalApiError('anthropic', <original message>)` instead of propagating raw.
- **`AnthropicClientModule`** (`backend/src/shared/anthropic-client/anthropic-client.module.ts`) — a `useFactory` provider for the `AnthropicClient` token, `exports: [AnthropicClient]`. Not imported into `AppModule` itself — a consuming feature module (e.g. Foundations Console) imports this module directly and injects `AnthropicClient`, never `RealAnthropicClient`/`FakeAnthropicClient` directly. Deliberately **not** [`fake-mode.md`](fake-mode.md)'s shared `fakeSwitchProvider()` helper — that helper covers the plain real/fake switch every other module needs, but the fake branch here needs one extra step: it sets `allowUnqueuedFallback = true` (see [`test-doubles.md`](test-doubles.md)) on the constructed `FakeAnthropicClient` instance, so a live, fake-mode-running app never 500s on an unscripted call the way a test-bound fake correctly still does. A test binds its own `FakeAnthropicClient` directly and never goes through this module, so it's unaffected.

## Using it

- Import `AnthropicClientModule` into a feature module and inject `AnthropicClient` — the same token `FakeAnthropicClient` already binds to in tests. `FAKE_MODE` governs which implementation is actually live in a running process; nothing feature-specific to configure.
- A streaming consumer (the first is Foundations Console's streaming `/messages` route) calls `streamMessage()` and iterates the yielded raw SDK events directly — no reshaping happens in this module.
- A failed real call always surfaces as `ExternalApiError('anthropic', ...)`, which [`api-error-handling.md`](api-error-handling.md)'s global filter shapes into the documented `502` response automatically for a normal request; a streaming consumer's own `try`/`catch` handles it via that doc's `shapeError()` instead, since a global filter can't intercept an already-streaming response.

## Testing

- `backend/src/shared/anthropic-client/real-anthropic-client.spec.ts` — `nock`-intercepted: constructs its SDK client from `AppConfigService.anthropicApiKey`, returns the exact shaped `Message` on success, yields raw stream events in order on a successful stream, returns the uploaded file id and sends the `files-api-2025-04-14` beta flag on `uploadFile()`, and rethrows a `createMessage()`, `streamMessage()`, or `uploadFile()` auth failure as `ExternalApiError('anthropic', ...)`.
- `backend/src/shared/anthropic-client/anthropic-client.module.spec.ts` — proves the module binds `FakeAnthropicClient` when `fakeMode` is `true` and `RealAnthropicClient` when `false`, and that the bound `FakeAnthropicClient`'s `allowUnqueuedFallback` is `true`, using a stub global config module (never the real `AppConfigModule`, so this test never loads a real `backend/.env`).
- `backend/test/anthropic-client.e2e-spec.ts` — a real, fully composed app (`AppConfigModule` + `AnthropicClientModule`) proving `RealAnthropicClient` resolves and makes a real, `nock`-intercepted SDK call. The `FAKE_MODE=true` binding path is deliberately not re-proven at this level — see [`testing-strategy.md`](../technical/testing-strategy.md) on why an env-toggling e2e test for a DI-switch binding doesn't reliably work and isn't needed once the unit-level module spec already covers it.
- `backend/src/testing/http-fixtures/anthropic.fixtures.ts` gained `mockAnthropicMessagesStream()`, a `nock` fixture for a streamed `POST /v1/messages` response — see [`test-doubles.md`](test-doubles.md).

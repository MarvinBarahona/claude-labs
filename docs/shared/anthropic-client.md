# Anthropic Client

A real backend adapter over the Anthropic SDK (`@anthropic-ai/sdk`), wired to the same `AnthropicClient` DI token the fake implementation already uses, so any feature that calls the Messages API for real injects `AnthropicClient` and gets the real client in real mode / the fake in fake mode with no per-feature branching.

## Interface

- **`AnthropicClient`** (`backend/src/shared/anthropic-client/anthropic-client.ts`) — the abstract-class DI token: `createMessage(params: AnthropicMessageParams): Promise<AnthropicMessage>` and `streamMessage(params): AsyncIterable<AnthropicStreamEvent>`, both typed directly against `@anthropic-ai/sdk`'s own `Message` / `MessageCreateParams` / `RawMessageStreamEvent` types (`AnthropicMessage` / `AnthropicMessageParams` / `AnthropicStreamEvent` are this file's own aliases for those). Lives at this build-included shared location — not under `backend/src/testing/` — because a production module needs to bind a real class to it; `FakeAnthropicClient` (see [`test-doubles.md`](test-doubles.md)) stays under `testing/` and imports the token from here.
- **`RealAnthropicClient`** (`backend/src/shared/anthropic-client/real-anthropic-client.ts`) — constructs `new Anthropic({ apiKey: config.anthropicApiKey })` via `AppConfigService` (no `maxRetries` override, unlike [`key-health.md`](key-health.md)'s deliberately-fast check — a real user-facing call gets the SDK's normal retry behavior). `createMessage()` calls `client.messages.create({ ...params, stream: false })`; `streamMessage()` calls `client.messages.create({ ...params, stream: true })` and yields the SDK's own raw stream events unmodified. Any thrown Anthropic SDK error (auth failure, rate limit, network error, etc.) is caught and rethrown as [`api-error-handling.md`](api-error-handling.md)'s `ExternalApiError('anthropic', <original message>)` instead of propagating raw.
- **`AnthropicClientModule`** (`backend/src/shared/anthropic-client/anthropic-client.module.ts`) — `providers: [fakeSwitchProvider<AnthropicClient>(AnthropicClient, { real: RealAnthropicClient, fake: FakeAnthropicClient })]`, `exports: [AnthropicClient]`. Not imported into `AppModule` itself — a consuming feature module (e.g. Foundations Console) imports this module directly and injects `AnthropicClient`, never `RealAnthropicClient`/`FakeAnthropicClient` directly. The `fakeSwitchProvider` generic is pinned explicitly to `AnthropicClient` rather than left to infer — see [`fake-mode.md`](fake-mode.md)'s "Using it" for why (a `private` field on `RealAnthropicClient` that `FakeAnthropicClient` lacks breaks TypeScript's inference otherwise).

## Using it

- Import `AnthropicClientModule` into a feature module and inject `AnthropicClient` — the same token `FakeAnthropicClient` already binds to in tests. `FAKE_MODE` governs which implementation is actually live in a running process; nothing feature-specific to configure.
- A streaming consumer (the first is Foundations Console's streaming `/messages` route) calls `streamMessage()` and iterates the yielded raw SDK events directly — no reshaping happens in this module.
- A failed real call always surfaces as `ExternalApiError('anthropic', ...)`, which [`api-error-handling.md`](api-error-handling.md)'s global filter shapes into the documented `502` response automatically for a normal request; a streaming consumer's own `try`/`catch` handles it via that doc's `shapeError()` instead, since a global filter can't intercept an already-streaming response.

## Testing

- `backend/src/shared/anthropic-client/real-anthropic-client.spec.ts` — `nock`-intercepted: constructs its SDK client from `AppConfigService.anthropicApiKey`, returns the exact shaped `Message` on success, yields raw stream events in order on a successful stream, and rethrows both a `createMessage()` and a `streamMessage()` auth failure as `ExternalApiError('anthropic', ...)`.
- `backend/src/shared/anthropic-client/anthropic-client.module.spec.ts` — proves the module binds `FakeAnthropicClient` when `fakeMode` is `true` and `RealAnthropicClient` when `false`, using a stub global config module (never the real `AppConfigModule`, so this test never loads a real `backend/.env`).
- `backend/test/anthropic-client.e2e-spec.ts` — a real, fully composed app (`AppConfigModule` + `AnthropicClientModule`) proving `RealAnthropicClient` resolves and makes a real, `nock`-intercepted SDK call. The `FAKE_MODE=true` binding path is deliberately not re-proven at this level — see [`testing-strategy.md`](../technical/testing-strategy.md) on why an env-toggling e2e test for a DI-switch binding doesn't reliably work and isn't needed once the unit-level module spec already covers it.
- `backend/src/testing/http-fixtures/anthropic.fixtures.ts` gained `mockAnthropicMessagesStream()`, a `nock` fixture for a streamed `POST /v1/messages` response — see [`test-doubles.md`](test-doubles.md).

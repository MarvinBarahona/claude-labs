# Technical — Forms

Update as the system evolves.

- **Field labels are title case, not uppercase.** `text-xs font-bold tracking-wide text-foreground` — no `uppercase` class. Keeps a field label visually distinct from the all-caps role/status badges used elsewhere in a lab's UI, so the two don't compete for the same "shouting" visual weight. Applies to a `<label>`'s `<span>`, a `<fieldset>`'s `<legend>`, and any other field-level caption.

- **A checkbox's own label is bold like other field labels, and states its current value in parentheses.** e.g. `Stream Response (Yes)` / `Stream Response (No)`, using the same `text-xs font-bold tracking-wide text-foreground` class as every other field label — not a plain unstyled `<span>` sibling to the input. A checkbox's checked state is easy to miss at a glance (unlike a slider or a text field, whose current value is always legible); spelling it out in the label closes that gap without adding a separate status element.

- **A form mixing compact controls with a `<textarea>` splits into a 2-column grid at the `lg` breakpoint, rather than stacking every control full-width.** Compact controls (radio groups, a slider, a checkbox) sit in one column, width-capped (e.g. `lg:max-w-xs`) so they don't stretch to fill it; the `<textarea>` fills the other column at full height (`h-full`) so both columns end level. Below `lg`, everything stacks full-width in document order. A settings form with more than a couple of compact controls plus a multi-line field gets unwieldy as one long full-width stack; splitting it keeps the form scannable at a glance on a wide viewport while still degrading cleanly on narrow ones. Wrap the grid row in `flex flex-col lg:grid lg:grid-cols-2 lg:gap-x-8 lg:items-start` on the container, with each column's last stacked item getting its own `pb-4 lg:pb-0` so the stacked-mobile spacing doesn't leak into the grid layout.

- **A visual separator placed after a settings form gets its own top margin, independent of the preceding row's padding.** e.g. `mt-6 ... border-t border-border pt-4` rather than relying on a neighboring element's `pb-*` to create the gap. If the separator's spacing depended on an unrelated element's padding instead, that padding changing for an unrelated reason would silently close the gap.

- **`structured-output-console`** — a DTO field pairing a `class-validator` decorator with a type imported from another file (e.g. `@IsIn(MODEL_TIERS) modelChoice: ModelTier;`) trips TS1272 (`emitDecoratorMetadata` needs a real value to reference a cross-file string-literal-union type) unless that type is imported via `import type` specifically — a plain `import { X, Y } from '...'` compiles fine for everything *except* the decorated field. Neither `npm test` (`ts-jest` + `isolatedModules`) nor `npm run lint` (type-aware eslint) catches this; only `tsc` itself does (surfaced here by the dev container's own `nest start --watch`). Worth a coding-convention line in `docs/technical/` (or `nest-conventions`) once another instance of the pattern shows up: any DTO field like this must import its cross-file type with `import type`.

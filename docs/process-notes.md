# Process Notes

Append-only log of suggestions the drafting/build process can't apply itself: coding-convention/process changes, and changes to files it doesn't own (`README.md`, `CLAUDE.md`, other skills). Written during planning or graduation, never applied automatically. Reviewed and cleared manually, on your own schedule.

- `content-block-builder` — a build-time correction can force a dependency's already-graduated permanent doc (`docs/shared/*.md`) to be edited twice if that doc is updated mid-build instead of deferred to graduation: this build's plan called for updating `anthropic-client.md`/`test-doubles.md` in place once the new `uploadFile()` method was built, and doing so immediately (before `nest build` caught that the plan's literal SDK call shape didn't actually compile) meant writing the doc, then rewriting it again once the real implementation changed. Worth confirming, as a standing convention, that any permanent doc outside a work item's own plan file — not just README/CLAUDE.md/skill files — stays untouched mid-build and gets its update applied only at graduation, once the implementation is settled.

# Process Notes

Append-only log of suggestions the drafting/build process can't apply itself: coding-convention/process changes, and changes to files it doesn't own (`README.md`, `CLAUDE.md`, other skills). Written during planning or graduation, never applied automatically. Reviewed and cleared manually, on your own schedule.

- `workflow-gallery` — running two implementation tracks in parallel against the same bind-mounted dev stack can trip an unrelated service's health-check gate while the other track's source is mid-edit with a compile error: one track's `docker compose run --rm <service> ...` command still waits on every service in its `depends_on` chain, not just the one it's actually testing. Worth deciding whether a parallel track's own commands should default to skipping dependency health-gating when they don't need the other track's live service.
- `workflow-gallery` — a feature's plan can silently omit a required testing bucket if its to-do checklist isn't explicitly checked against the full current bucket list: this one's checklist covered backend unit/integration and frontend unit but never itemized its in-app lab doc or its browser-E2E spec as their own to-do items, so both were missing from the actual build until asked about directly. Worth confirming plan-writing always derives a feature's to-do checklist from the complete current bucket list, doc before E2E spec (an E2E spec needs a real doc already in place to assert against).
- `workflow-gallery` — README's claim that `npm run lint` closes the type-check gap `npm test` leaves open is now known to be incomplete: a genuine `tsc` diagnostic (TypeScript's weak-type-detection check on a generic constrained to an all-optional shape) passed both `npm test` and `npm run lint` with zero errors while the real dev container was crash-looping on it — only an actual `nest build`/`tsc --noEmit` catches this class (see `testing-strategy.md`, updated directly). Worth supplementing the documented two-command safety net with an explicit `tsc --noEmit` step, or narrowing the README claim to note the exception.
